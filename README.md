# 薬の飲み忘れ装置 ― 実装一式

「薬を自動で出す」のではなく、**飲み忘れを検知して家族に知らせる**ことに絞った安価な見守りピルケースの実装一式です。薬は市販の週間ピルケースに人がセットし、装置は「取り出したか・正しい区画か」を見張って記録・通知します。

構成: ラズパイ(カメラ) + ESP32(センサー) + Google スプレッドシート/Apps Script + Web。データベースは使いません。

---

## 0. このZIPの中身

```
pillbox/
  esp32/pillbox_esp32.ino       ESP32: 時間通知＋蓋開閉(取り出し)検知＋死活信号
  raspberrypi/pill_camera.py     ラズパイ: トレイ撮影→区画の空きを判定(画像は破棄)
  raspberrypi/config.example.json  カメラの区画範囲(ROI)などの設定例
  gas/Code.gs                    GAS: 記録＋飲み忘れメール＋死活監視
  web/index.html                 家族向けダッシュボード(服薬カレンダー)
```

## 2. セットアップ手順（順番どおり）

### A. スプレッドシートと GAS
1. Google スプレッドシートを新規作成。先頭シート1行目に見出しを入れる: `日時 / 送信元 / 区画 / 出来事 / 詳細`
2. 拡張機能 → Apps Script を開き、`gas/Code.gs` を貼り付け。
3. `SETTINGS.mailTo` を家族のメールに、`schedule` を服薬時刻に書き換える。
4. デプロイ → 新しいデプロイ → 種類「ウェブアプリ」→ アクセス「全員」。出てくる `…/exec` のURLを控える。
5. 「トリガー」画面で `checkMissed` と `checkHeartbeat` を**時間主導型・5分おき**に登録する。

### B. ESP32
1. Arduino IDE に ESP32 ボードを追加(ESP32-C3 等を選択)。
2. `esp32/pillbox_esp32.ino` を開き、Wi-Fi情報・`GAS_URL`・`COMPARTMENTS`(ピンと時刻)を設定。
3. 各区画にリードスイッチ＋磁石、ブザー、LED、補充ボタンを配線(下記)。
4. 書き込み。シリアルに `send event=... code=200` が出れば送信成功。

**配線の目安(ESP32-C3)**
```
区画リードスイッチ(各区画)  : 片側→GND、もう片側→GPIO(2,3,4…) ※INPUT_PULLUP使用
  蓋を閉めると磁石が近づきON(LOW)、開けるとOFF(HIGH)
ブザー                     : GPIO5 → ブザー → GND
LED                        : GPIO6 →(330Ω)→ LED → GND
補充モードボタン           : GPIO7 → ボタン → GND
電源                       : USB 5V(常時通電)
```

### C. ラズパイ（カメラ）
1. ラズパイに OpenCV と requests を入れる: `pip install opencv-python requests`（Pi Camera を使うなら `picamera2` も）
2. `config.example.json` を `config.json` にコピーし、`gas_url` と各区画の `roi`(四角い範囲)を設定。
   - `roi` は `[左x, 上y, 幅w, 高さh]`(ピクセル)。`--debug` で撮った `debug_last.jpg` を見ながら合わせると楽。
3. 動作確認: `python3 pill_camera.py --debug`
4. 服薬時刻の少し後に自動実行(cron 例: 朝の確認を 8:35 に):
   ```
   35 8 * * * /usr/bin/python3 /home/pi/pillbox/raspberrypi/pill_camera.py
   ```

### D. Web ダッシュボード
1. スプレッドシートを「リンクを知っている全員が閲覧可」に共有。
2. `web/index.html` の `SHEET_ID` を、シートURLの `/d/` と `/edit` の間の文字列に置き換える。`SCHEDULE` は ESP32/GAS と必ずそろえる。
3. ブラウザで開く(または Netlify Drop に置く)と、今日の状態・直近7日・出来事が表示される。

## 3. 見落としやすい点（先に押さえる）

1. **これは医療機器ではありません。** 同種の市販品も薬機法上の医療機器ではない位置づけです。診断・治療効果はうたわず、「飲み忘れ防止の補助」として使ってください。
2. **「取り出した」≠「飲んだ」。** 本装置が確認できるのは取り出しまでで、実際に服用したことの証明はできません。
3. **補充モードを必ず使う。** 家族が薬をセットし直すときは補充ボタンを押してから。押さないと取り出しと誤判定します。
4. **通知の信頼性。** ネット断・停電の「無通知」が最大のリスクです。本実装は死活監視(heartbeat)で「沈黙」も検知してメールします。
5. **カメラは照明・置き方で精度が変わります。** トレイの位置と明るさを一定に保ち、`fill_threshold` を現場で調整してください。簡易方式のため、確実性が要るなら錠剤識別(拡張)の学習が必要です。
6. **時刻設定の一致。** ESP32 の `COMPARTMENTS`、GAS の `schedule`、Web の `SCHEDULE` は必ず同じ内容にそろえてください。

## 4. 拡張のヒント

- **錠剤の識別(飲み間違い防止):** 区画ROIの画像を自分で集めて学習(TensorFlow Lite 等)し、`pill_camera.py` の判定を差し替える。推論はラズパイ内で行い、画像は残さない方針を維持。
- **現在状態を Cloudflare KV に:** アクセスが増えてスプレッドシート読み取りを軽くしたい場合、最新状態だけ KV に上書きし、Web はまず KV を読む(履歴グラフのみスプレッドシート)。
- **LINE通知:** GAS の `sendMail` を LINE Messaging API 等に差し替える。
